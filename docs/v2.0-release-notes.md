# AI Kit v2.0.0 Release Notes

## 2026-04-09

### Summary
**Breaking change**: All skills and agents are now namespaced with a `kit-` prefix to prevent collisions with global/built-in skills in Claude Code, Cursor, and other AI tools. This is a major version bump from v1.x. Existing users should run `ai-kit update` — old unprefixed files are automatically cleaned up.

### Why This Change?

Claude Code resolves skills at multiple scopes: project-level overrides global-level when names collide. Without namespacing:
- AI Kit's project-level `/review` silently overrides a user's personal global `/review`
- Near-duplicates like `/security` (global) and `/security-check` (AI Kit) both appear in suggestions, causing confusion
- Agent names like `architect` or `code-reviewer` conflict with Claude Code's built-in agent types

The `kit-` prefix eliminates all three problems. Every AI Kit skill (`/kit-review`) and agent (`@kit-planner`) is instantly recognizable and never collides with personal or built-in names.

### Migration Guide

**For existing projects:**
```bash
npx @mikulgohil/ai-kit update
```

This single command:
1. Backs up your current configs
2. Removes old unprefixed skill/agent files (`.claude/skills/review/`, `.claude/agents/planner.md`, etc.)
3. Generates new `kit-` prefixed files
4. Updates `ai-kit.config.json` with the new names

**For new projects:** No action needed — `ai-kit init` generates `kit-` prefixed names from the start.

### New Features

#### `SKILL_PREFIX` constant
The `kit-` prefix is defined as a single constant (`SKILL_PREFIX` in `src/constants.ts`). All skill and agent names are derived from base names + this prefix at runtime. Changing the namespace in the future requires editing only this one constant.

#### Automatic cleanup on update
`ai-kit update` now detects and removes old unprefixed skill directories, legacy command files, and agent files before writing new ones. This prevents the exact duplication problem the namespace was designed to solve.

### Security Fix

#### `exec()` replaced with `execFile()` in token dashboard
- **File**: `src/commands/tokens.ts`
- **Issue**: `exec()` passed a file path through a shell, which could interpret metacharacters in directory names
- **Fix**: Switched to `execFile()` which invokes the binary directly with an argv array — no shell interpretation

### Decisions
- **`kit-` chosen over `ai-kit-`** — shorter, memorable, doesn't collide with any known tool's naming convention
- **Prefix is a constant, not configurable** — configurable prefixes add complexity without clear demand. Can be made configurable later if needed
- **Dead `commands.ts` deleted** — was never imported after `skills.ts` superseded it. Keeping it created a false impression of being maintained
- **Cleanup runs before file generation** — old files are removed first, then new kit-prefixed files are written, ensuring a clean state
