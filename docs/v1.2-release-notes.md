# AI Kit v1.2.0 Release Notes

## 2026-03-18

### Summary
Major feature release adding hooks, agents, context modes, session management, multi-agent orchestration, and security auditing — inspired by patterns from `everything-claude-code` but tailored specifically for Next.js/React/Sitecore workflows.

### New Features

#### Hooks System (Automated Quality Checks)
- Auto-format on file edit (Prettier or Biome, auto-detected)
- TypeScript type-check after `.ts/.tsx` edits
- Console.log warning detection
- Git push safety reminder
- Three hook profiles: minimal, standard, strict
- Generated in `.claude/settings.local.json`

#### Specialized Agents (8 agents)
- `kit-planner` — implementation planning
- `kit-code-reviewer` — deep quality review
- `kit-security-reviewer` — OWASP Top 10, XSS, CSRF
- `kit-e2e-runner` — Playwright tests (conditional: only if Playwright installed)
- `kit-build-resolver` — build/type error resolution
- `kit-doc-updater` — documentation sync
- `kit-refactor-cleaner` — dead code cleanup
- `kit-sitecore-specialist` — Sitecore XM Cloud (conditional: only if Sitecore detected)

#### Context Modes (3 modes)
- `dev` — focus on building features
- `review` — focus on quality and security
- `research` — read-only exploration and analysis

#### Session Management Skills (3 new skills)
- `/kit-save-session` — persist session state
- `/kit-resume-session` — restore previous session
- `/kit-checkpoint` — verification snapshot

#### Orchestration Skills (3 new skills)
- `/kit-orchestrate` — multi-agent coordination
- `/kit-quality-gate` — comprehensive quality checks
- `/kit-harness-audit` — AI configuration health check

#### Security Audit CLI Command
- `ai-kit audit` — checks for secrets, hook validity, agent config, MCP security
- Outputs A-F health grade with score

#### Biome Detection
- Scanner now detects Biome as a formatter
- Hooks auto-configure based on whether Prettier or Biome is found

### Decisions
- Hook profiles (minimal/standard/strict) mirror the existing strictness levels but are independent — you can have relaxed rules with strict hooks
- Agents are conditionally generated based on scan results — kit-e2e-runner only if Playwright, kit-sitecore-specialist only if Sitecore
- Hooks generate in `settings.local.json` (not `settings.json`) to avoid conflicting with project-level settings
- Biome is detected alongside Prettier; hooks use whichever is found (Biome preferred)
- All new features are scoped to Next.js/React/Sitecore — no Python, Go, Rust, etc.
