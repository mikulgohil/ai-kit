# AI Kit v2.4 — Next.js 16 Readiness Spec

**Status**: Draft for review
**Author**: Mikul Gohil (with Claude)
**Date**: 2026-05-21
**Target release**: v2.4 (minor)

---

## Why this exists

Next.js 16 GA + 16.2 (shipped 2026-05-08) introduced material changes that
make AI Kit's current Next.js guidance stale or actively wrong:

- `middleware.ts` is deprecated in favor of `proxy.ts` (Node runtime, not Edge).
- `"use cache"` directive + `cacheComponents` config replace
  `experimental.ppr` and `experimental.dynamicIO`.
- `revalidateTag(tag, cacheLife)` now requires a second argument.
- New `updateTag()` and `refresh()` Server Action APIs.
- `params`, `searchParams`, `cookies()`, `headers()`, `draftMode()` are async.
- React 19.2 in App Router: View Transitions, `useEffectEvent`, `<Activity>`.
- React Compiler stable; `reactCompiler: true` opt-in.
- Turbopack default bundler; FS cache opt-in via
  `experimental.turbopackFileSystemCacheForDev`.
- `next lint` removed. `images.domains` deprecated. Parallel routes require
  `default.js`. `images.dangerouslyAllowLocalIP` defaults to off.
- Node.js 20.9+ and TypeScript 5.1+ are minimums.

Additionally, Next.js 16 ships a **Next.js DevTools MCP** server that gives
AI agents (Claude Code, Cursor, etc.) routing/caching/rendering insight,
unified browser+server logs, automatic error stack traces, and page
awareness. This is high-value to auto-wire from AI Kit.

Audit of current ai-kit code confirms concrete stale sites
(see "Audit results" below).

---

## Goal

Make AI Kit emit correct, current guidance for Next.js 16/16.2 projects and
expose new Claude Code primitives (Next.js DevTools MCP, Monitor tool,
session-title hooks) so AI Kit-configured projects get the best debugging
and agentic experience automatically.

## Scope (in)

### A. Scanner extensions — `src/scanner/nextjs.ts`

Extend `NextjsResult` with version-gated detections of Next.js 16 features:

- `proxyFile: 'proxy' | 'middleware' | 'both' | 'none'`
- `cacheComponents: boolean`
- `reactCompiler: boolean`
- `turbopackConfigLocation: 'top-level' | 'experimental' | 'none'`
- `turbopackFsCache: boolean`
- `parallelRoutesMissingDefault: string[]` (route paths missing `default.js`)
- `imageConfigUsesDomains: boolean` (deprecated form)
- `reactVersion: string | undefined`
- `reactMajorMinor: number | undefined` (for 19.2 gating)
- `nodeEnginesSatisfiesNext16: boolean | undefined`

Implementation: regex-based parsing of `next.config.{js,ts,mjs,mts}` with
version-gate (only run extended parse when `nextjsMajorVersion >= 16`).
No new runtime dependencies. Document false-positive scenarios in a JSDoc
comment block on the parser.

### B. Template rewrites

- `templates/claude-md/nextjs-app-router.md` — full rewrite covering:
  - `proxy.ts` (Node runtime) as new primary; `middleware.ts` Edge legacy.
  - `"use cache"` directive + `cacheComponents` programming model.
  - `revalidateTag(tag, cacheLife)` two-arg signature.
  - `updateTag()` Server Action read-your-writes.
  - `refresh()` for uncached data.
  - Async `params`/`searchParams`/`cookies()`/`headers()`/`draftMode()`.
  - View Transitions, `useEffectEvent`, `<Activity>`.
  - Turbopack default + FS cache experimental flag.
  - `images.remotePatterns` over `images.domains`; `localPatterns`;
    `maximumRedirects`; `dangerouslyAllowLocalIP`.
  - Removals: AMP, `next lint`, `experimental.ppr`, `experimental.dynamicIO`,
    `experimental.turbopack`, `serverRuntimeConfig`/`publicRuntimeConfig`.
- `templates/cursorrules/nextjs-app-router.md` — parallel changes.
- Both templates: structure as two sections — "Next.js 16+ patterns" (active)
  and "Next.js 15 legacy" (only emitted when major < 16) — so a single
  template can serve both. Template selector chooses section based on
  scanner output.

### C. Skill updates — `commands/`

- Add `kit-proxy.md` — new skill for `proxy.ts` (Node runtime) creation /
  modification. Includes Sitecore preview mode, auth, i18n, redirect
  recipes against `proxy.ts`.
- Keep `kit-middleware.md` as a thin alias that:
  - For `next>=16`: prints deprecation note and delegates to `kit-proxy`.
  - For `next<16`: behaves as today.
- `kit-server-action.md` — replace single-arg `revalidateTag` examples with
  `revalidateTag(tag, 'max')` + add `updateTag()` and `refresh()` recipes.
- `kit-new-page.md` — emit `await params` / `await searchParams` in
  generated code for Next.js 16+ targets.
- `kit-fix-bug.md` — when Next.js DevTools MCP is present (see § D), bias
  the skill to reference it as the preferred debugging context source.

### D. MCP emitter — new file `src/generator/mcp.ts`

Currently AI Kit has only an MCP **scanner** (`src/scanner/mcp.ts`), not
an emitter. Create a writer with these properties:

- Reads existing `.mcp.json` (project root) and `.claude/settings.json`.
- Merges idempotently — never overwrites user-defined MCP server entries
  with the same key.
- Adds `next-devtools` MCP entry when `nextjsMajorVersion >= 16` is detected.
- Uses the canonical Next.js DevTools MCP command (TBD: confirm from
  Next.js docs at <https://nextjs.org/docs/app/guides/mcp> during
  implementation).
- Records `_generated_by: ai-kit` alongside any entries it added, so
  `ai-kit update` and `ai-kit rollback` can manage them.
- Wired into `ai-kit init`, `ai-kit update`, `ai-kit migrate`.
- Also extend `src/scanner/mcp.ts` to recognize the DevTools MCP entry on
  re-scan.

Default behavior: **auto-wire when `next>=16` detected** (decided
2026-05-21). Future flag: `ai-kit init --no-mcp-autowire` if user requests
opt-out (out of scope for v2.4; track in ROADMAP).

### E. Hook recipe extensions — `src/generator/hooks.ts`

Add the following hook recipes (gated on Claude Code CLI version where
possible, but emit unconditionally for `strict` profile):

- `UserPromptSubmit` → `hookSpecificOutput.sessionTitle` — auto-name
  sessions by repo name + intent prefix.
- Monitor-tool-driven recipes:
  - "Watch CI" — emit hook that pipes Monitor tool events for the active
    PR's CI run into the conversation.
  - "Watch dev server" — pipe `next dev` output as transcript messages.
- Status line: emit `refreshInterval` (default 30s) and reference
  `workspace.git_worktree` in the JSON input so the status line shows
  worktree context.
- Compose with newer slash commands: at session start, hint that
  `/team-onboarding` and `/autofix-pr` are available when the project has
  CI configured.

### F. Health / Doctor checks — `commands/kit-health` & doctor codepath

New Next.js 16-aware checks:

- ❌ Node engine < 20.9 with `next>=16` (error).
- ❌ TypeScript < 5.1 with `next>=16` (error).
- ⚠️ `middleware.ts` present without `proxy.ts` on `next>=16` (warning).
- ❌ Parallel routes without `default.js` on `next>=16` (error).
- ⚠️ `images.domains` used (deprecation warning).
- ⚠️ `experimental.turbopack` / `experimental.ppr` / `experimental.dynamicIO`
  found in `next.config` (deprecation warning).
- ℹ️ Next.js DevTools MCP not wired (info — surfaced as a "run `ai-kit
  update`" hint).

### G. Migration aid — `kit-next16-migrate` skill

New skill at `commands/kit-next16-migrate.md` that:

- Detects upgrade-eligible projects (`next<16`).
- Recommends running `npx @next/codemod@canary upgrade latest`.
- After upgrade, runs `ai-kit update` to refresh CLAUDE.md / .cursorrules /
  `.mcp.json` to v2.4 templates.
- Walks the user through:
  - Renaming `middleware.ts` → `proxy.ts` (when applicable).
  - Updating `revalidateTag` calls.
  - Awaiting `params`/`searchParams`/`cookies()`/`headers()`.
  - Adding missing `default.js` to parallel routes.
  - Removing `serverRuntimeConfig` / `publicRuntimeConfig`.

### H. Tests — `tests/`

- `tests/scanner/nextjs.spec.ts` — fixtures for each new detection
  (cacheComponents, reactCompiler, proxy/middleware variants, parallel
  routes with/without default.js, images.domains).
- `tests/generator/mcp.spec.ts` — idempotence: writing twice produces
  identical output; existing user entries preserved; rollback restores.
- `tests/generator/claude-md.spec.ts` — Next.js 16 template selection
  and content snapshots.

### I. Release notes — `docs/v2.4-release-notes.md`

Mirror the structure of `v2.0-release-notes.md` with:

- Summary of changes.
- Migration table (Old vs New API).
- Upgrade path: `ai-kit update` for existing v2.3 users.
- Known limitations (regex parser false positives, etc.).
- Link to this spec.

## Non-goals (out)

- Tailwind v4 — no release in window; defer to a separate spec.
- Sitecore XM Cloud / Optimizely SaaS CMS changes — no release in window.
- Cursor / Windsurf / Aider / Cline rule format changes — none in window.
- AGENTS.md cross-tool standardization — no progress in window.
- AST-based `next.config` parsing — defer to v2.5+ if regex proves brittle.
- Removing Next.js 15 support — dual-path is required.
- New CLI subcommand (`ai-kit upgrade --next16`) — the
  `kit-next16-migrate` skill is sufficient for v2.4; a real CLI command
  can come later.

## Success criteria

- [ ] `ai-kit init` on a fresh Next.js 16.2 project produces a CLAUDE.md
      whose Next.js section references `proxy.ts`, `"use cache"`,
      `updateTag`, `refresh()`, and async params/searchParams.
- [ ] Same `ai-kit init` produces zero references to deprecated APIs
      (`middleware.ts` as primary, single-arg `revalidateTag`, sync
      `params`/`searchParams`, `images.domains`).
- [ ] `.mcp.json` is written automatically with a `next-devtools` MCP
      entry when `next>=16` is detected; existing user entries preserved.
- [ ] `ai-kit health` reports at least three new Next.js 16-aware checks
      (Node engine, parallel-routes `default.js`, `proxy.ts` presence).
- [ ] `kit-middleware` slash command still works on `next<16` and shows
      a deprecation notice with link to `kit-proxy` on `next>=16`.
- [ ] All new code paths have vitest coverage; existing tests still pass.
- [ ] `ai-kit update` on a v2.3 project shows a clean diff and auto-backup;
      `ai-kit rollback` restores cleanly.
- [ ] v2.4 release notes published in `docs/`.

## Constraints

- No new runtime dependencies. Current deps: `@inquirer/prompts`, `chalk`,
  `commander`, `fs-extra`, `ora`.
- Tests via vitest; lint via eslint; build via tsup.
- Stack scope: Sitecore + Next.js + Tailwind + TypeScript + monorepo tools.
  Do not extend to other frameworks.
- Skill prefix `kit-` preserved.
- `ai-kit update` must remain idempotent and auto-backup.
- Per memory: top-down adoption model — copy/messaging should still target
  tech leads mandating it for teams.

## Open questions / assumptions

1. **Canonical Next.js DevTools MCP install command** — verify exact
   command and any required Node/PNPM/NPM args from the official docs at
   <https://nextjs.org/docs/app/guides/mcp> during implementation. Spec
   assumes a stable `npx @next/devtools-mcp` style invocation.
2. **Claude Code CLI version gating** — the Monitor tool requires Claude
   Code v2.1.98+; status line `refreshInterval` requires v2.1.94+. For
   v2.4 we emit unconditionally and document the floor; we do not
   actively detect Claude Code version.
3. **kit-middleware alias mechanism** — current AI Kit skill loader does
   not have an "alias" concept. Implementation will be a thin wrapper
   skill file that delegates. Need to confirm the loader handles this
   cleanly; otherwise we may need a small loader change.
