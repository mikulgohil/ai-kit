### 7. Safe `update` Command (Merge Strategy)
- [ ] Current problem: `update` overwrites any manual edits to CLAUDE.md
- [ ] Solution: Wrap generated content in markers `<!-- AI-KIT:START -->` ... `<!-- AI-KIT:END -->`
- [ ] On update: only replace content between markers, preserve everything outside
- [ ] Alternative: show diff before overwriting, ask for confirmation

### 8. Documentation Threshold
- [ ] Current rule: every component needs a `.docs.md` — too aggressive for simple components
- [ ] Add threshold: doc file required for components > 50 lines OR > 3 props
- [ ] Simple wrappers and primitives: JSDoc comment is sufficient
- [ ] Update both `base.md` templates (claude-md + cursorrules)
- [ ] Update `/new-component` command output rules

### 9. Generated Output Versioning
- [ ] Add ai-kit version comment at top of generated files: `<!-- Generated by ai-kit v1.0.0 -->`
- [ ] Store version in `ai-kit.config.json`
- [ ] On `update`: detect version mismatch, warn if major version differs
