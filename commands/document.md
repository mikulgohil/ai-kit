# Document

Generate documentation for an existing component, module, or utility.

## What This Command Does

Reads an undocumented (or poorly documented) file and generates a comprehensive `.docs.md` file with props table, usage examples, edge cases, and a change log. Also adds JSDoc comments to exported functions.

## How to Use

```
/document src/components/ProductCard/ProductCard.tsx
```

For a utility:

```
/document src/lib/formatPrice.ts
```

## What Gets Generated

### 1. Component Documentation File

For a component like `ProductCard.tsx`:

**Generated: `ProductCard.docs.md`**

```markdown
# ProductCard

Displays a product with image, title, price, and add-to-cart action.
Used in product listing pages and search results.

## Props

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| product | Product | Yes | — | Product data object |
| variant | 'default' \| 'compact' | No | 'default' | Display variant |
| onAddToCart | (id: string) => void | No | — | Called when user clicks add to cart |
| showBadge | boolean | No | false | Show sale/new badge if applicable |

## Usage

### Basic

```tsx
<ProductCard product={product} />
```

### With add-to-cart handler

```tsx
<ProductCard
  product={product}
  onAddToCart={(id) => addToCart(id)}
/>
```

### Compact variant (for sidebar/widgets)

```tsx
<ProductCard product={product} variant="compact" />
```

## Behavior

- Displays placeholder image if `product.image` is null
- Truncates title to 2 lines with ellipsis
- Formats price using project's `formatPrice` utility
- Badge shows "Sale" if `product.salePrice` exists, "New" if added within 7 days

## Edge Cases

- Missing image: Shows gray placeholder with product icon
- Long title: Truncated with CSS `line-clamp-2`
- Price of 0: Displays "Free" instead of "$0.00"
- Out of stock: Disables add-to-cart button, shows "Out of Stock" text

## Dependencies

- `@/lib/formatPrice` — price formatting
- `next/image` — optimized image rendering
- `@/components/Badge` — sale/new badge

## Change Log

- 2026-03-17: Initial documentation generated by ai-kit
```

## Questions (asked before documenting)

1. What file do you want documented?
2. Is there any context not obvious from the code? (design decisions, business rules)
3. Are there known edge cases or gotchas?
4. Who is the audience? (team devs, external consumers, future maintainers)

## Rules

- **Read the code first** — understand before documenting
- **Document the WHY, not just the WHAT** — props tables are obvious, design decisions aren't
- **Include real examples** — copy-pasteable, not pseudocode
- **Note edge cases** — what happens with null, empty, zero, very long strings?
- **Add doc reference comment** to the component: `// Docs: ./ComponentName.docs.md`
- **Only create doc files for complex components** — >50 lines or >3 props
- **For simple utilities** — JSDoc comment is sufficient, no separate doc file
